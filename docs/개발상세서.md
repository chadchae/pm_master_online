# Project Manager V2 — 개발상세서

## 1. 프로젝트 개요

### 목적
~/Projects/ 폴더 내 모든 프로젝트, 문서, 노트, 이슈, 학습, 가이드라인을 웹에서 운영·관리하는 로컬 프로젝트 매니저.

### 핵심 가치
- **파일시스템 직접 동기화** — DB 없이 실제 폴더/파일과 1:1
- **연구 + 개발 하이브리드** — 칸반 sublabel로 연구 워크플로우 반영
- **Claude Code 연동** — 작업지시 → 임베디드 터미널 → 자동 체크리스트
- **개인 허브** — 비밀번호만으로 접근, 1인 사용 최적화

### 기술 스택
| 구성 | 기술 |
|------|------|
| Backend | FastAPI (Python 3.12) |
| Frontend | Next.js 15, React 19, Tailwind CSS |
| MD Editor | @uiw/react-md-editor |
| Terminal | @xterm/xterm + WebSocket PTY |
| Auth | bcrypt + 파일 기반 토큰 |
| 메타데이터 | YAML frontmatter (pyyaml) |
| i18n | 커스텀 hook (ko/en, 280+ 키) |
| 포트 | Backend 8002, Frontend 3002 |

## 3. 기능 상세

### 3.1 인증 시스템
- 비밀번호만 (아이디 없음), bcrypt 해시
- 파일 기반 토큰 (`data/tokens.json`) — 서버 재시작 시 유지
- 토큰 만료: 24시간
- 쿼리 파라미터 토큰 지원 (다운로드용)
- 비밀번호 변경 시 모든 토큰 무효화

### 3.2 대시보드
- **칸반 보드**: Initiation/Discussion → Development/Data & Lit Review → Testing/Analysis → Completed/Writing → Archived/Submitted
- **리스트 뷰**: 테이블 + 컬럼 소트 (이름/단계/유형/포트/수정일)
- **뷰 전환**: Kanban / List 토글
- **유형 필터**: 체크박스 (연구/개발/연구+개발)
- **카드 순서**: 같은 컬럼 내 드래그 → `card_order.json` 저장
- **카드 정보**: label, 폴더명, description, 메타태그 아이콘, 진행률 바, 목표종료일, 관계자
- **카드 액션**: 수정(→상세)/다운로드(zip)/삭제(→trash), hover 시 표시
- **프로젝트 이동**: 드래그 → 모달(작업지시 입력) → 실제 폴더 이동
- **활성 프로젝트**: 유형별 카운트 (연구: N | 개발: N | 기타: N)

### 3.3 Ideas 페이지
- `1_idea_stage` 프로젝트만 카드 그리드로 표시
- Promote to Initiation (작업지시 모달 연동)
- Discard → 7_discarded
- 새 아이디어 생성 (폴더명/표시이름/설명/유형)

### 3.4 프로젝트 상세 (6탭)

#### Documents 탭
- 파일 목록 + MD 에디터 (@uiw/react-md-editor, 분할 뷰)
- 폴더 drill-down (경로 바 + 뒤로가기)
- +New 드롭다운: 파일 / 폴더
- 다중 선택 삭제, 개별 삭제
- 파일/폴더 아이콘 구분

#### Instructions 탭
- 수동 작업지시 생성 (텍스트 + 커스텀 체크리스트)
- `docs/작업지시_YYYY-MM-DD.md` 생성
- 같은 날 → 시간 스탬프로 append
- 기본 체크리스트: "Update work instruction status upon completion"

#### Todo 탭 (칸반)
- 3컬럼: 할 일 / 진행 중 / 완료
- 체크박스 → 즉시 done/undo
- 카드: 제목, 설명, 우선순위 뱃지, 담당자, 마감일, 생성일/완료일
- 드래그앤드롭 (컬럼 간 + 컬럼 내)
- 인라인 편집, 삭제

#### Issues 탭 (스레드)
- 상태: Open / In Progress / Resolved / Closed
- 우선순위: Low / Medium / High / Critical
- 라벨 (태그), 담당자
- 필터 버튼 + 각 상태별 카운트
- 펼치기 → 전체 설명 + 상태 변경 + 댓글 스레드
- 댓글 CRUD (수정/삭제 hover 버튼)
- 이슈 수정 (인라인 폼)
- Resolve 버튼 (resolved_at 자동)

#### Schedule 탭
- **테이블 뷰**: 제목(서브태스크 인덴트), 시작일/종료일/기간/담당자/상태/진행률/선행작업, 상태 인라인 변경, 수정/삭제
- **간트 차트**: CSS/SVG 기반 (라이브러리 없음)
  - 날짜 축 (월/일), 주말 음영
  - 태스크 바 (색상=상태, 길이=기간, 진행률 표시)
  - 오늘 마커 (빨간 점선)
  - 마일스톤 ◆ 다이아몬드
  - 선행작업 화살표 (SVG line + arrowhead)
  - 카테고리별 트랙 (색상 헤더 + 그룹)
  - 반응형 dayWidth (1W/2W/3W/1M/All)
- **마일스톤**: 날짜 + 설명 + 연결 태스크
- **카테고리**: 이름 + 색상, 신규 생성, 간트 트랙 시각화
- **선행작업 강제**: 미완료 시 in_progress 잠금 (🔒)
- **Overdue 감지**: end_date < today → 빨간 표시

#### Settings 탭
- **Project Information**: 경로, 단계 (읽기 전용)
- **Tags & Priority**: 유형, 포트, 중요도(★1-5), 위급도, 긴급도, 협업, 주도, 오너, Related People
- **Timeline & Progress**: 목표종료일, 실제종료일, 오늘
- **서브태스크**: CRUD + 드래그 순서 + Done/Cancel 체크 + 진행률 바 연동

### 3.5 헤더 요약 위젯
- Todo: done/total + 진행률 바 + todo/wip 카운트
- Issues: open/total + open/done 카운트
- Schedule: 실제 데이터 (planned/in_progress/done/overdue)

### 3.6 사이드바 패널

#### Quick Note
- `_notes/_temp/`에 즉시 메모 저장
- 5개 카테고리로 이동/정리 (연구아이디어/호기심/사색/기술/개인)
- 삭제

#### Work Execution
- 미완료 작업지시 스캔
- Launch Claude Code → 임베디드 터미널 (xterm.js + WebSocket PTY)
- 프롬프트에 "완료 후 체크리스트 업데이트" 자동 포함
- 세션 종료 시 목록 새로고침

#### Work Status
- 전체 프로젝트 작업현황 대시보드
- 프로젝트별 진행률, 체크리스트 상세
- 전체 진행률 바

### 3.7 글로벌 기능

| 기능 | 설명 |
|------|------|
| People | 관계자 카드 (이름/소속/역할/전문분야/관계), connections, 자동생성 |

## 6. 데이터 흐름

### 프로젝트 메타데이터
```
_아이디어노트.md (YAML frontmatter)
  ↓ scanner_service.scan_projects()
  ↓ _parse_yaml_frontmatter() + _parse_readme() fallback
  ↓ subtask_service.get_counts() 주입
  → API /api/projects → Frontend cards
```

### 작업지시 흐름
```
칸반 드래그 → MoveProjectModal (작업지시 입력)
  → POST /api/projects/move (서버 중지 → 폴더 이동 → 잔여물 정리)
  → create_transition_note() → docs/작업지시_YYYY-MM-DD.md
  → Work Execution 패널 → Launch Claude Code
  → Claude Code가 체크리스트 [x] 업데이트
  → Work Status에서 확인
```

### Description 동기화
```
프로젝트 상세에서 description 수정
  → PUT /api/projects/{name}/description
  → _아이디어노트.md 업데이트
  → README.md 업데이트 (blockquote)
  → _토의록.md 업데이트 (> 요약: 라인)
```
