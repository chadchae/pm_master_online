## 2026-03-19 (2) 핵심 기능 구현

**시간**: 오후 15:00 ~ 18:00 (CST)
**주제**: 전체 기능 구현

### 구현 내역

1. **기본 구조**: 비밀번호 인증, 파일시스템 스캐너, 사이드바 네비게이션
2. **칸반 + 리스트 뷰**: Kanban/List 전환, 드래그앤드롭 단계 이동
3. **Ideas 페이지**: 아이디어 전용, Promote/Discard
4. **메타태그**: YAML frontmatter 전환 (15개 파일), 중요도/위급도/긴급도/협업/오너
5. **진행률**: 서브태스크 + 4등분 바, 목표/실제 종료일
6. **문서 관리**: MD 에디터, 다중 선택 삭제, 새 파일 생성
7. **작업지시**: 단계 전환 자동 생성, 수동 생성, Claude Code 연동
8. **Work Execution**: 임베디드 터미널 (xterm.js + WebSocket PTY)
9. **Work Status**: 전체 프로젝트 작업현황 대시보드
10. **People**: 관계자 카드, connections, Related People 태그
11. **Quick Note**: _temp/ 연동, 카테고리 이동
12. **Trash**: 복원/영구삭제

### 아키텍처 결정

- DB 없음 — 파일시스템 직접 스캔 (JSON 저장: auth, tokens, people, card_order)
- YAML frontmatter 표준화 (blockquote에서 마이그레이션)
- Description 변경 시 다중 파일 동기화 (_아이디어노트 + README + _토의록)
- 작업지시 프롬프트에 "체크리스트 업데이트하라" 자동 포함 (자동완료 대신)
- 파일 기반 토큰 (서버 재시작 시 유지)

### 발견된 이슈

- API 응답 wrapper 불일치 ({projects: [...]} vs [...])
- _find_project_path 잔여 폴더 문제 (이동 후 빈 폴더 남음)
- auth.json 디버깅 중 리셋 반복
- onSessionEnd 중복 호출 (toast 스팸)
- _아이디어노트.md 없는 프로젝트에서 description 수정 실패
